## 目录说明

目录结构稍有调整

```
Assets
├── 3rdPaty             // 第三方插件
├── AssetArt            // 美术资源目录
├── AssetBundle         // 打包资源目录，目录下的资源会自动按目录打包
│   ├── Audios        	// 音频资源
│   ├── AutoGen        	// 导出的配置和协议，二进制格式
|       └── GameConfig  // 导出的二进制配置, .bytes
│   ├── Configs        	// 其它配置
│   ├── DLL           	// HybridCLR AOT需要的dll文件，同时也包括Game相关热更新文件
│   ├── Fonts          	// 字体文件
│   ├── Scenes         	// 游戏场景文件
│   ├── UI           	// UI Prefab目录
│   └── Texture         // 图片资源
│       ├── RawImage       // RawImage资源
│       ├── SpriteImage    // SpriteImage资源
│       │   ├── Atlas       // 需要自动生成图集的UI素材目录
│       └── ├── Raw         // 不需要自动生成图集的UI素材目录
├── AssetRaw            // TEngine的资源目录，可以删除
├── Editor              // 编辑器脚本目录
├── HybridCLRData       // hybridclr相关目录
├── AssetBundle         // 打包资源目录，目录下的资源会自动按目录打包
│   ├── Scenes        	// 启动场景目录，只包括 Launcher.unity
├── Scripts             // 程序集目录
|   ├── AutoGenProto	// GameProto.dll程序集，自动导出的配置和协议的结构文件，AssetBundle/AutoGen中的是数据
|   ├── GameCore		// GameCore.dll程序集，游戏框架逻辑，通用模块，Tools等
|   ├── Runtime			// GameRuntime.dll程序集，游戏业务逻辑，Scene, UI等
|   |   ├── GameData	// 业务数据定义，不是配置数据
|   |   ├── Scene		// Scene相关
|   |   ├── UI			// UI逻辑
|   └── Procedure		// 默认的Assembly-CSharp.dll程序集，游戏启动阶段不同过程逻辑
└── TEngine             // 框架核心目录              
    ├── AssetSetting    // YooAsset资源设置  
    ├── Editor          // TEngine-Editor程序集
    └── Runtime         // TEngine-Runtime程序集
```

- AssetArt
  - 为原始美术资源，所有美术的原始资源都应该放在这里，不会直接打到包中，但如果未被项目使用的资源，特别是较大的资源，不要放这里，会影响项目的打开速度，无关资源应该放在专门的美术仓库中管理
  - 最终程序使用的资源应该是prefab，asset，spriteatlas等文件类型
  - 场景、prefab、图集中引用到的原始美术文件可以放在这里
  
  



## 运行模式

不同运行模式对应的适用情况说明，完整说明见官方文档：[初始化 | YooAsset](https://www.yooasset.com/docs/guide-runtime/ResourceInit)

- EditorMode
  - 开发模式专用，资源修改后立即生效，不用构建资源包，只在Editor下生效
  - 如果资源多之后，会因为启动时的自动模拟构建引起卡顿，可以通过直接指定已有的清单路径来避免每次重复执行模拟构建。换句话说，就是如果有必要，也可以改为手动构建资源包的方式来运行，来避免资源多之后的自动构建卡顿问题。
- OfflineMode
  - 会从StramingAsset下加载资源，需要构建资源包
- HostPlayMode
  - 会从远端获取资源清单，并进行更新操作，需要构建资源包
- WebPlayMode
  - 针对WebGL专属模式，比如微信小游戏、抖音小游戏选用此模式，需要构建资源包

特别要说明的时，在编辑器下可以直接修改工程顶部工具栏中的选项来进行运行模式的切换，但打成exe,apk这种必须要需要修改Boot工程中的`GameEntry/ResourceDriver`来切换，此项常规设为 HostPlayMode。



如果启用更新时，还需要在UpdateSettings里，设置ProjectName和ResDownLoadPath，最终会合成一条访问URL：

相应的版本信息获取方式：```http://127.0.0.1:8081/DemoTEngine/Android/MainPackage.version```

1. 前面部分是域名，更新设置最关系的内容
2. DemoTEngine，为工程名称，对应ProjectName
3. Android，为平台名称
4. 平台目录下放置更新文件
